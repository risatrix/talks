What Programming Feels Like

What We Did

  1. Created a Github repository called my-page
  2. cloned it onto Desktop
    git clone [your git address here]
  3. added and commited an index.html file to it
    git add .
    git commit -m 'first commit'
  4. pushed the code back to Git hub
    git push origin master
  5. created a branch called gh-pages
    git checkout -b gh-pages
  6. pushed to gh-pages (=deploying code)
    git push origin gh-pages
    url = [yourusername].github.io/my-page

Git Concepts

Generally, we're tracking and deploying code changes via specific steps.

git status  shows any files changed
git add  adds files to staging area
git commit  names and commits the set of changes
git push  moves the new commit to Github

a commit ≈ a chapter of code history, a note to your future self and others about the what and why of changes.

Note: committing does not push the code to Github, it only names the change. Pushing completes the deployment.
